# 天下足球需求文档

## 1. 项目概述

### 1.1 项目名称

天下足球

### 1.2 项目定位

天下足球是一个围绕足球历史资料、球员能力评分、国家队/俱乐部荣誉、位置分布与综合排行的数据管理和展示系统。

系统第一阶段以《紫禁之巅.lakesheet》为核心数据来源，将当前表格中的球员、国家、俱乐部、荣誉规则和统计结果结构化，形成可查询、可筛选、可统计、可维护的 Web 应用。

### 1.3 项目目标

- 将分散在 lakesheet 表格中的足球数据沉淀为结构化数据库。
- 支持球员、国家、俱乐部、荣誉规则、评分体系的统一管理。
- 支持按足联、国家、俱乐部、位置、年代、PA/CA、球员类型、荣誉分等维度查询和排行。
- 支持世界概览、国家汇总、俱乐部汇总、球员汇总等看板展示。
- 支持后续从 lakesheet、Excel 或 CSV 导入数据，并保留校验、预览和错误回滚能力。

### 1.4 第一版数据来源

当前已读取的《紫禁之巅.lakesheet》包含以下工作表：

| 工作表 | 作用 |
| --- | --- |

| 俱乐部汇总 | 俱乐部维度的球员能力分布、位置分布和统计数据 |
| 俱乐部荣誉 | 俱乐部洲际赛事、世俱杯、国内赛事等荣誉明细 |

## 2. 用户与使用场景

### 2.1 目标用户

- 数据维护者：负责录入、修正和审核球员、国家、俱乐部、荣誉规则等数据。
- 足球资料浏览者：查看球员、球队、国家队、荣誉和排行榜。
- 数据分析者：按不同维度比较球员、俱乐部、国家或足联实力分布。
- 管理员：负责用户权限、数据导入、规则配置和系统维护。

### 2.2 核心场景

- 用户想查询某位球员的基础资料、PA、主要位置、国籍、效力球队和荣誉情况。
- 用户想查看巴西、意大利、英格兰等国家的历史球员数量、PA 分布和国家队荣誉。
- 用户想比较皇家马德里、巴塞罗那、国际米兰、尤文图斯等俱乐部的球员分布和荣誉分。
- 用户想按 ST、AMC、MC、GK 等位置查看历史最佳球员或候选名单。
- 管理员想导入新的 lakesheet 数据，并在导入前检查重复 UID、空值、格式错误和统计口径变化。
- 管理员想调整荣誉分值或赛事系数，并重新计算国家、俱乐部、球员荣誉分。

## 3. 功能需求

### 3.1 首页 / 数据看板

首页用于展示项目整体数据概览。

#### 主要内容

- 球员总数、国家数量、俱乐部数量、荣誉规则数量。
- PA 区间分布：190、180、170、160、150、140、130、120、110、100。
- 足联分布：欧足联、南美足联、中北美足联、亚足联、非足联、大洋足联。
- 位置分布：ST、AML、AMC、AMR、MC、DMC、DL、DC、DR、GK 等。
- 荣誉排名摘要：国家荣誉榜、俱乐部荣誉榜、球员荣誉榜。
- 最近导入记录、最近数据更新时间、待处理备注数。

#### 页面状态

- loading：展示骨架屏或加载状态。
- empty：无数据时提示需要导入数据。
- error：统计数据加载失败时展示重试入口。
- permission：无查看权限时展示权限提示。

#### 球员列表

字段包括：

- UID
- 中文名
- 英文名
- 出生日期
- 过世日期
- 年龄
- 主要位置
- 可踢位置
- CA
- PA
- 身高
- 体重
- 球衣号码
- 肤色
- 发色
- 种族
- 左/右脚
- 球员类型
- 足联
- 国家 UID
- 国籍
- 效力球队数
- 年代

#### 列表能力

- 支持关键词搜索：中文名、英文名、UID。
- 支持筛选：足联、国家、位置、PA 区间、CA 区间、年代、球员类型、是否过世。
- 支持排序：PA、CA、年龄、球队数、荣誉分、冠军数。
- 支持分页。
- 支持保存筛选条件。

#### 球员详情

详情页包括：

- 基础资料卡片。
- 能力信息：CA、PA、PA 区间、身价映射。
- 位置信息：主要位置、所有位置、位置分类。
- 国籍与足联信息。

- 综合荣誉分。
- 相关备注和修订记录。

#### 最佳阵容

对应 `最佳` 工作表。

- 按位置展示最佳球员候选。
- 支持每个位置维护候选排序。
- 支持阵型视图，例如 4-3-3、4-4-2、3-5-2。
- 支持管理员编辑最佳阵容候选。

- 不同联赛或地区系数，例如英意西德、法、阿巴、荷葡、中其他。

### 3.5 荣誉与评分体系

荣誉与评分体系对应 `荣誉`、`国家荣誉`、`俱乐部荣誉`、`球员荣誉`。

#### 荣誉规则管理

管理员可维护：

- 荣誉类别：国家、俱乐部、个人。
- 荣誉层级：国际、洲际、一般杯赛、其他杯赛、附加分。
- 适用足联：欧足联、南美足联、中北美足联、亚足联、非足联、大洋足联。
- 奖项名称。
- 基础分值。
- 名次权重：冠军、亚军、季军、殿军。
- 地区或赛事系数。
- 是否参与总分计算。
- 生效状态。

#### 评分计算

系统需要支持：

- 按规则计算国家荣誉分。
- 按规则计算俱乐部荣誉分。
- 按规则计算球员荣誉分。
- 支持手动触发重算。
- 支持规则变更后批量重算。
- 支持查看计算明细，避免只展示最终结果。

#### 已知评分口径

- 国家荣誉中，冠军、亚军、季军、殿军存在不同权重。
- 洲际赛存在足联系数差异，例如欧洲、亚洲、非洲、南美、中北美、大洋洲。
- 俱乐部赛事存在洲际级别和国内赛事系数。
- 球员荣誉中包含国际奖、洲际奖、国家队荣誉、俱乐部荣誉、国内联赛、国内杯赛等维度。

### 3.7 数据导入与导出

#### 导入来源

- lakesheet 文件。
- Excel 文件。
- CSV 文件。

#### 导入流程

1. 上传文件。
2. 解析文件结构。
3. 识别工作表。
4. 映射字段。
5. 校验数据。
6. 展示导入预览。
7. 用户确认导入。
8. 写入数据库。
9. 触发统计和荣誉分重算。
10. 生成导入报告。

#### 校验规则

- UID 不允许重复。
- 必填字段不能为空。
- 数字字段必须为合法数字。
- 日期字段必须可解析。
- PA、CA、身高、体重等字段应在合理范围内。
- 公式单元格需要读取公式结果值。
- 链接字段需要保留文本和 URL。
- 未识别工作表需要提示并跳过。

#### 失败恢复

- 导入失败时不得污染正式数据。
- 支持事务回滚。
- 支持下载错误报告。
- 支持只导入某些工作表。

### 3.8 备注与修订管理

备注模块对应 `备注` 工作表。

#### 功能

- 维护待处理问题。
- 维护外部参考链接。
- 标记完成状态。
- 记录处理人和处理时间。
- 关联球员、国家、俱乐部或荣誉规则。

#### 示例事项

- 球员评分待定。
- 评分更改记录。
- 外部榜单参考。
- 最佳阵容参考。

## 4. 前端需求

### 4.1 页面结构

第一版建议包含：

- 首页看板
- 球员列表
- 球员详情
- 国家列表
- 国家详情
- 俱乐部列表
- 俱乐部详情
- 荣誉规则管理
- 最佳阵容
- 数据导入
- 备注管理
- 系统配置

### 4.2 通用交互

- 表格支持分页、筛选、排序、列显示配置。
- 筛选项支持清空、重置、保存。
- 详情页支持从列表返回，并保留筛选状态。
- 编辑表单支持校验、错误提示、防重复提交、提交成功提示和失败恢复。
- 导入流程支持进度展示、错误报告和确认弹窗。

### 4.3 状态要求

所有核心页面需要考虑：

- loading：加载中。
- empty：无数据。
- error：加载失败或保存失败。
- disabled：不可操作状态。
- permission：无权限状态。
- success：保存、导入、重算成功。

## 5. 后端需求

### 5.1 核心能力

- 用户认证与权限校验。
- 球员、国家、俱乐部、荣誉规则的 CRUD。
- 列表查询、详情查询、组合筛选、排序和分页。
- 统计聚合：PA 分布、位置分布、足联分布、荣誉分排行。
- 荣誉分计算和重算。
- 文件导入、导出和导入报告。
- 操作日志和数据修订记录。

### 5.2 推荐模块划分

- Auth 模块：登录、用户、角色、权限。
- Player 模块：球员资料、球员汇总、球员荣誉。
- Country 模块：国家资料、国家荣誉、国家统计。
- Club 模块：俱乐部资料、俱乐部荣誉、俱乐部统计。
- Honor 模块：荣誉规则、分值体系、计算引擎。
- Dashboard 模块：世界概览和排行榜。
- Import 模块：文件解析、字段映射、校验、导入报告。
- Config 模块：位置、类型、潜力、能力值、身价映射。
- Remark 模块：备注、待办、参考链接。

## 7. API 需求

### 7.1 球员接口

- `GET /api/players`：球员列表，支持分页、搜索、筛选、排序。
- `GET /api/players/:id`：球员详情。
- `POST /api/players`：创建球员。
- `PUT /api/players/:id`：更新球员。
- `DELETE /api/players/:id`：删除球员。
- `GET /api/players/rankings`：球员排行榜。
- `GET /api/players/:id/honors`：球员荣誉明细。

### 7.2 国家接口

- `GET /api/countries`：国家列表。
- `GET /api/countries/:id`：国家详情。
- `GET /api/countries/:id/honors`：国家荣誉明细。
- `GET /api/countries/rankings`：国家排行榜。

### 7.3 俱乐部接口

- `GET /api/clubs`：俱乐部列表。
- `GET /api/clubs/:id`：俱乐部详情。
- `GET /api/clubs/:id/honors`：俱乐部荣誉明细。
- `GET /api/clubs/rankings`：俱乐部排行榜。

### 7.4 荣誉接口

- `GET /api/honor-rules`：荣誉规则列表。
- `POST /api/honor-rules`：创建荣誉规则。
- `PUT /api/honor-rules/:id`：更新荣誉规则。
- `DELETE /api/honor-rules/:id`：删除荣誉规则。
- `POST /api/honors/recalculate`：触发荣誉分重算。
- `GET /api/honors/calculation-detail`：查看计算明细。

### 7.5 看板接口

- `GET /api/dashboard/overview`：首页概览。
- `GET /api/dashboard/world-summary`：世界概览统计。
- `GET /api/dashboard/pa-distribution`：PA 分布。
- `GET /api/dashboard/position-distribution`：位置分布。
- `GET /api/dashboard/confederation-distribution`：足联分布。

### 7.6 导入接口

- `POST /api/import/tasks`：上传并创建导入任务。
- `GET /api/import/tasks/:id`：查看导入任务详情。
- `POST /api/import/tasks/:id/preview`：生成导入预览。
- `POST /api/import/tasks/:id/confirm`：确认导入。
- `POST /api/import/tasks/:id/cancel`：取消导入。
- `GET /api/import/tasks/:id/errors`：下载错误报告。

## 8. 权限需求

### 8.1 角色

- 游客：可查看公开数据。
- 登录用户：可查看完整数据和使用高级筛选。
- 编辑者：可新增、编辑、导入数据。
- 管理员：可管理用户、权限、规则、导入和重算。

### 8.2 权限点

- 查看球员资料。
- 查看国家资料。
- 查看俱乐部资料。
- 查看荣誉明细。
- 编辑基础资料。
- 编辑荣誉规则。
- 执行数据导入。
- 执行评分重算。
- 删除数据。
- 管理用户和角色。

## 9. 非功能需求

### 9.1 性能

- 列表接口默认分页，避免一次返回大量数据。
- 常用统计数据应支持缓存。
- 荣誉分重算可异步执行，并展示任务状态。
- 首页看板接口应在 2 秒内返回。

### 9.2 数据可靠性

- 导入流程必须支持事务。
- 规则变更和重算需要记录版本。
- 关键数据删除应软删除。
- 支持导入前备份和导入失败回滚。

### 9.3 可维护性

- 荣誉规则不应硬编码在业务代码中，应由配置表维护。
- 位置、类型、潜力、PA 区间等基础枚举应可配置。
- 统计口径需要文档化，并在页面展示说明。

### 9.4 安全

- 所有写操作必须鉴权。
- 导入文件需要限制大小和类型。
- 导入内容需要做格式校验，避免异常公式或恶意内容。
- 操作日志需要记录用户、时间、动作和影响对象。
