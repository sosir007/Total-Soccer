# 标签颜色规范

本文档记录前端当前的主题色、基础 UI 色和业务标签色。目标是让页面新增颜色时有明确归属，不再在页面里散落 `#xxxxxx`、`rgb()` 或临时变量。

## 设计原则

- 业务标签统一使用 `SemanticTag`，不要在页面里新增散落的十六进制颜色。
- 标签色统一采用 `text / border / background` 三段式 token，保证浅色标签在表格和详情页里都清晰。
- 基础文字色统一使用四档 token：重点、常规、弱提示、白色反白。
- 图标型名次（冠军、亚军、季军、殿军、四强）继续使用图片图标，同时使用专属文字强调色，不改成纯色标签。
- Element Plus 的按钮、弹窗和基础反馈色可以保留；业务语义标签优先使用项目组件。
- 天机阁「色彩体系」页面只读展示当前前端 token，不直接编辑颜色、不写入数据库。

## 颜色来源

| 类型 | 维护位置 | 说明 |
| --- | --- | --- |
| 全局 CSS token | `apps/web/src/styles/main.scss` | 页面背景、文字、边框、阴影、业务标签实际色值 |
| 业务颜色映射 | `apps/web/src/utils/tag-theme.ts` | `SemanticTag` variant、图表 palette、颜色分组、标签说明 |
| 只读查看页 | 天机阁「色彩体系」 | 从 CSS 变量读取当前实际色值 |

> 注：新增颜色时优先补 `main.scss` token，再补 `tag-theme.ts` 映射和本文档说明。

## 一、基础 UI 颜色

基础 UI 色用于页面骨架、正文、边框、底色和阴影。它们不表达具体业务含义，因此不要拿足联色、名次色、PA/CA 色来替代。

### 1.1 基础文字色

| 用途 | 变量 | 色值 |
| --- | --- | --- |
| 重点文字、标题、重要数值、详情关键值 | `--text-color-primary` | `#222222` |
| 默认正文、表格正文、详情普通内容 | `--text-color-regular` | `#666666` |
| 弱提示、说明、副标题、UID、辅助时间 | `--text-color-secondary` | `#999999` |
| 深色背景反白文字 | `--text-color-white` | `#ffffff` |

使用约定：

- 新增页面和组件统一使用 `--text-color-primary / regular / secondary / white`。
- 不再新增 `--ink`、`--muted` 这类短别名。
- 除主题色、基础文字色和业务语义标签色外，不在页面里新增其他文字色；确需特殊颜色时先补充规范。

### 1.2 基础界面色

| 类型 | 用途 | 变量 | 色值 |
| --- | --- | --- | --- |
| 页面 | 页面兜底背景 | `--color-page-background` | `#f7faf6` |
| 底色 | 默认内容白底 | `--color-surface-default` | `#ffffff` |
| 底色 | 柔和信息底色 | `--color-surface-muted` | `#fbfdf9` |
| 底色 | 柔和内容底色 | `--color-surface-soft` | `#f7fbf7` |
| 底色 | 次级柔和底色 | `--color-surface-subtle` | `#f8fbf7` |
| 底色 | 品牌柔和底色 | `--color-surface-brand-soft` | `#edf8f1` |
| 底色 | 暖色 hover 底色 | `--color-surface-warm-soft` | `#faf7ec` |
| 边框 | 默认边框/分割线 | `--color-border-default` | `#e2eae1` |
| 边框 | 弱边框 | `--color-border-muted` | `#dfe8df` |
| 边框 | 更弱边框 | `--color-border-subtle` | `#eef3ed` |

| 阴影 | 品牌聚焦阴影 | `--shadow-focus-brand` | `0 8px 20px rgba(31, 139, 85, 0.1)` |

使用约定：

- 页面背景、卡片底色、弹窗底色、通用分割线、下拉浮层优先使用上表 token。
- 不再新增 `--bg`、`--line`、`--shadow`、`--soft-shadow` 这类短别名。
- `--color-border-brand-*` 只用于和品牌主色相关的 hover、focus、selected 状态。

### 1.3 品牌与强调色

| 用途 | 变量 | 色值 |
| --- | --- | --- |
| 品牌主色、主按钮、主题进度条 | `--color-brand-primary` | `#1f8b55` |
| 金色强调、品牌辅助强调 | `--color-accent-gold` | `#c89436` |

使用约定：

- 主题主色只用于品牌、主操作和少量强调，不用作普通成功色或“是/否”类状态色。
- 金色强调用于品牌辅助和重点装饰，不替代冠军名次色；冠军色使用 `placement-champion`。

## 二、业务标签颜色

业务标签色由 `_tokens.scss` 中的 `--tag-*-text / border / bg` 提供实际色值，由 `tag-theme.ts` 负责映射到页面 variant。

### 2.1 业务维度总览

| 模块 | 值 |
| --- | --- |
| 基础 | 中性、是、否 |
| 足联 | AFC、UEFA、CONMEBOL、CAF、CONCACAF、OFC |
| 位置 | 前场、中场、后场、门将；ST、AML、AMR、AMC、MC、DMC、ML、MR、WBL、WBR、DL、DC、DR、GK |
| 赛事分类 | 国际、洲际、国内、其他 |
| 赛事级别 | 一级、二级、三级、四级 |
| 状态 | 启用、停用、现行、停办、纳入统计、排除统计、现役、退役、在世、已故、成功、失败 |
| 对象类型 | 国家队、俱乐部、球员、赛事、奖项 |

### 2.2 足联

- token 前缀：`confed-*`
- 适用页面：国家列表、球员列表、球队列表、国家详情、荣誉矩阵、选择器 chip。
- 设计目标：六大足联要能在同一张列表里快速区分，避免和赛事分类、对象类型完全撞色。

基础布尔值约定：

- `boolean-yes` 与 `boolean-no` 使用靛蓝和朱橙冷暖互补色，避免复用主题绿。
- 页面出现明确的“是/否”值时统一使用该组标签，不借用启用、成功、顶级奖等其他业务状态色。

### 2.3 位置

- token 前缀：`position-*`
- 位置色分为两层：前场、中场、后场、门将是大类组色，具体位置使用接近所属组色但不重复的颜色。
- 前场保持红橙体系：ST、AML、AMR 分别向深红、玫红、橙色区分。
- 中场保持绿青体系，但 AMC 使用琥珀色突出前腰的进攻属性；MC、DMC、ML、MR、WBL、WBR 在绿、青、天蓝之间展开。
- 后场保持蓝靛体系：DL、DC、DR 使用蓝、靛蓝、青蓝区分。
- 门将保持紫色体系：GK 使用比门将组色略深的紫色。

### 2.4 赛事分类与赛事级别

- 赛事分类 token 前缀：`competition-category-*`
- 赛事级别 token 前缀：`competition-level-*`
- 适用页面：赛事列表、赛事详情、荣誉明细、荣誉规则。
- 赛事分类表达范围，赛事级别表达重要性，两者不要复用同一套颜色。

### 2.5 状态与对象类型

- 状态 token 前缀：`status-*`
- 对象类型 token 前缀：`object-*`
- 状态色用于启用/停用、纳入/排除、现役/退役、在世/已故、成功/失败等二元或结果类展示。
- `现行/停办`只表达赛事生命周期；已经停办的历史赛事仍可保持启用并纳入荣誉统计。
- 对象类型色用于国家队、俱乐部、球员、赛事、奖项等对象识别；目前主要用于赛事管理对象列和色彩体系展示。

## 三、名次与能力颜色

名次和能力属于高频业务识别信息，可以比普通标签更醒目，但仍然只做文字、边框、浅背景的轻量强调。

### 3.1 名次颜色

| 名次 | 颜色变量 | 文字色 |
| --- | --- | --- |
| 冠军 | `placement-champion` | `#d4a017` |
| 亚军 | `placement-runner-up` | `#64748b` |
| 季军 | `placement-third-place` | `#b87333` |
| 殿军 | `placement-fourth-place` | `#2f80c7` |
| 四强 | `placement-semi-finalist` | `#5b5f97` |

使用约定：

- 名次展示仍然优先使用已有图片 icon。
- 表头、统计数字、年份列表等文字强调走 `placement-*` 色。
- 不用主题绿突出冠军，避免和品牌主色、成功状态混在一起。

### 3.2 PA / CA 分级

| 范围 | 文案 | 颜色变量 | 主色 |
| --- | --- | --- | --- |
| 190-200 | 传奇 | `ability-legend` | `#f53f3f` |
| 180-189 | 顶级 | `ability-elite` | `#d91ad9` |
| 170-179 | 巨星 | `ability-superstar` | `#722ed1` |
| 160-169 | 一流 | `ability-first-class` | `#165dff` |
| 150-159 | 优秀 | `ability-excellent` | `#3491fa` |
| 140-149 | 主力 | `ability-starter` | `#14c9c9` |
| 130-139 | 轮换 | `ability-rotation` | `#00b42a` |
| 120-129 | 潜力/合格 | `ability-prospect` | `#7bc616` |
| 111-119 | 普通 | `ability-normal` | `#d99904` |
| 100-110 | 基础 | `ability-basic` | `#d25f00` |
| 小于 100 / 空值 / 非法值 | 未分 | `ability-muted` | `#4e5969` |

使用约定：

- 第一版只用于 PA / CA，不用于总 PA、平均 PA、荣誉分、身价。
- 空值、非法值、小于 100 统一走 `ability-muted`。

## 四、图表颜色

图表色服务于 ECharts 的 series palette，用于柱状图、折线图、饼图等数据可视化。它不属于业务标签，不使用 `text / border / background` 三段式。

### 4.1 图表调色板

| 顺序 | 名称 | 变量 | 色值 | 用途 |
| --- | --- | --- | --- | --- |
| 1 | 鲜蓝 | `--chart-series-blue` | `#5470c6` | 默认主序列、核心指标 |
| 2 | 明绿 | `--chart-series-green` | `#91cc75` | 第二序列、正向或增长数据 |
| 3 | 亮金 | `--chart-series-gold` | `#fac858` | 第三序列、重点对比数据 |
| 4 | 珊瑚 | `--chart-series-coral` | `#ee6666` | 第四序列、变化或警示数据 |
| 5 | 天蓝 | `--chart-series-sky` | `#73c0de` | 第五序列、结构分布 |
| 6 | 翠绿 | `--chart-series-emerald` | `#3ba272` | 第六序列、补充分类 |
| 7 | 橙色 | `--chart-series-orange` | `#fc8452` | 第七序列、暖色补充 |
| 8 | 紫色 | `--chart-series-violet` | `#9a60b4` | 第八序列、多分类补充 |
| 9 | 粉紫 | `--chart-series-pink` | `#ea7ccc` | 第九序列、醒目补充 |
| 10 | 灰紫 | `--chart-series-mauve` | `#6e7079` | 第十序列、中性数据 |

### 4.2 使用约定

- ECharts 图表优先使用 `chartPalette`，不要在图表 option 里直接写散落色值。
- 多分类图使用 `chartPalette.categorical`，少量分类使用 `chartPalette.compactCategorical`。
- 单序列主图使用 `chartPalette.primary`，次级单序列图使用 `chartPalette.accent`。
- 图表色可以和业务标签色有少量接近，但不承担业务语义；同一页面同时出现标签和图表时，以标签语义为准。
- CSS token 用于文档和「色彩体系」展示；ECharts 实际 option 使用 `tag-theme.ts` 中的十六进制值，避免 canvas 场景无法解析 CSS 变量。

## 五、第三方与兼容变量

### 5.1 Element Plus 主题接入

- `--el-text-color-primary / regular / secondary` 指向项目基础文字色。
- `--el-color-primary` 指向 `--color-brand-primary`。
- Element Plus 的 light / dark 派生变量暂时保留手写值，后续如要进一步收敛，可单独做主题派生工具。

### 5.2 兼容别名

- `--confed-*` 和 `--position-*` 简短变量用于兼容旧组件。
- 新增页面优先使用 `SemanticTag` 或 `--tag-*-text / border / bg`，不要继续扩展简短别名。

## 六、暂不收敛的颜色

### 6.1 场景化视觉色

- hero、球场、紫禁之巅、自定义渐变背景等低复用视觉色，当前允许保留在局部样式里。
- 这类颜色不是基础 UI 色，也不强行抽成全局 token，避免 token 数量失控。

## 七、后续可扩展

- 色彩体系：后续可以新增 `baseColor + generated` 模式，自动派生浅背景和边框；当前保留人工维护的三色 token，避免影响可读性和现有视觉。
- 荣誉分：建议按真实数据分布单独设置区间，不复用 PA/CA。
- 总 PA、平均 PA：建议用于国家/俱乐部榜单时单独设置排行型颜色。
- 身价、年龄、数量：暂不分级，避免颜色语义过载。
