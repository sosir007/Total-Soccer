# 数据库备份与恢复

本项目的 GitHub 仓库只保存代码、Prisma migration 和文档。你在系统里手动录入的国家、俱乐部、球员、赛事、奖项等业务数据保存在本机 PostgreSQL 数据库中，不会随 `git push` 自动上传到 GitHub。

## 备份

建议在大量录入数据、执行 migration、批量导入或重算前手动备份一次。

```bash
mkdir -p backups
pg_dump "$DATABASE_URL" -Fc -f backups/total-soccer-$(date +%Y%m%d-%H%M%S).dump
```

如果没有在当前终端设置 `DATABASE_URL`，可以直接使用项目默认连接串：

```bash
pg_dump "postgresql://postgres:postgres@localhost:5432/total_soccer?schema=public" -Fc -f backups/total-soccer.dump
```

## 恢复

恢复前请确认目标数据库是你要覆盖的数据库。恢复会把备份中的数据写回 PostgreSQL。

```bash
pg_restore --clean --if-exists --no-owner --dbname "$DATABASE_URL" backups/total-soccer.dump
```

恢复后建议执行：

```bash
pnpm prisma:migrate
pnpm --filter @total-soccer/api prisma:generate
```

## 建议习惯

- 每次长时间手动录入前先备份。
- 每次执行 migration 前先备份。
- 重要备份文件可以复制到 iCloud、移动硬盘或其他私有存储。
- 不建议把 `.dump` 数据库备份文件直接提交到 GitHub。
